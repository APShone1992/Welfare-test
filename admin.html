🚀 How to set up (free, 3 minutes)

  1. Go to supabase.com and click Start your project (sign in with GitHub)
  2. Click New project, give it a name like welfare-support, choose a region close to you, set a database password
  3. Wait ~2 minutes for it to build, then go to SQL Editor and run this SQL to create the tables:
    create table sms_log ( id bigint generated always as identity primary key, ts bigint not null, sender_name text default '', phone text default '', type text default '', content text default '' ); create table unresolved ( id bigint generated always as identity primary key, ts bigint not null, query text default '', type text default '' ); create table sessions ( id bigint generated always as identity primary key, ts bigint not null, date text default '', messages int default 0 ); alter table sms_log enable row level security; alter table unresolved enable row level security; alter table sessions enable row level security; create policy "public read" on sms_log for select using (true); create policy "public insert" on sms_log for insert with check (true); create policy "public delete" on sms_log for delete using (true); create policy "public read" on unresolved for select using (true); create policy "public insert" on unresolved for insert with check (true); create policy "public delete" on unresolved for delete using (true); create policy "public read" on sessions for select using (true); create policy "public insert" on sessions for insert with check (true);
  4. Go to Settings → API, copy your Project URL and anon/public key
  5. Paste both below and click Save & Test
  6. Paste the same values into index.html in the two lines at the top of the Supabase Bridge script
